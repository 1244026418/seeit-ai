# SeeIt 会议复盘

使用 SeeIt AI MCP 工具将会议视频整理成可执行且可追溯的复盘。

## 工作流

1. 确定 `media_id`；缺失时调用 `list_media`，候选不唯一时请求确认。
2. 调用 `get_video_metadata`，确认视频状态和证据是否可用。
3. 分别调用 `search_video_evidence` 检索：
   - “决定、确认、结论、通过、否决”等决策表达；
   - “待办、负责、完成、截止、跟进”等行动表达；
   - “风险、阻塞、依赖、争议、未解决”等问题表达。
4. 对每项候选调用 `get_evidence_window`，检查说话内容的前因后果。
5. 合并重复事项；按时间顺序保留最直接的证据。用 `verify_video_citations` 校验所有最终引用。

## 输出要求

输出会议概览、已确认决策、行动项、风险与未决事项。行动项包含事项、负责人、截止时间、状态和证据时间戳。

负责人、截止时间或决定状态没有被视频明确说出时写“未明确”，不得根据语气、职位或上下文猜测。将讨论中的提议与正式决定区分开。

## 分析任务

用户要求生成并保存完整复盘时，调用 `start_video_analysis`，目标应明确包含决策、行动项、负责人、风险和证据。用 `get_analysis_status` 查询状态，完成后调用 `get_analysis_report`。任务未完成时返回任务 ID，不要连续高频轮询。
