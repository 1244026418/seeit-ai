# SeeIt 证据审计

使用 SeeIt AI MCP 工具逐条验证报告结论，不以报告本身作为事实来源。

## 工作流

1. 确定 `media_id` 和待审计报告。未提供报告时调用 `get_analysis_report`；未提供视频时调用 `list_media` 并确认候选。
2. 将报告拆成可独立验证的事实性结论，提取每条结论中的时间戳、来源和引用文本。
3. 对已有引用批量调用 `verify_video_citations`。
4. 对校验失败或没有引用的结论调用 `search_video_evidence`，使用结论中的核心实体和动作作为查询词。
5. 对新命中调用 `get_evidence_window`，检查上下文是否真正支持结论，而不只是包含相同关键词。
6. 将每条结论标记为“支持”“部分支持”“不支持”或“无法判断”。

## 判定规则

- 支持：时间戳、来源和语义均与原始证据一致。
- 部分支持：证据只支持结论的一部分，或结论包含视频未说明的扩大解释。
- 不支持：引用不存在、时间戳错误、语义相反或仅有关键词重合。
- 无法判断：ASR/OCR 缺失、内容截断或证据不足。

## 输出要求

先给出审计概览，再输出逐条审计表：结论、状态、原引用、核验依据、问题和建议修订。最后列出支持率及需人工复核项。

不得因为结论听起来合理就判定为支持。不得把外部知识当作视频证据。建议修订只能缩小到现有证据明确支持的范围。
